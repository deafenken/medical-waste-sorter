# RK3588 NPU 移植指南

> 把项目从树莓派迁移到 RK3588（如 Orange Pi 5、Radxa Rock 5、香橙派 5 Plus
> 等）：硬件 NPU 推理速度比 Pi 5 CPU 快 5–8 倍，YOLOv8n 可以稳定跑 30+ FPS。

---

## RK3588 vs 树莓派 5（YOLOv8n 实测参考）

| 平台 | 后端 | 输入 640×640 | FPS | 备注 |
|---|---|---|---|---|
| Pi 5 8GB | PyTorch CPU | 640 | 2-4 | 别在生产用 |
| Pi 5 8GB | NCNN CPU | 640 | 8-12 | 默认推荐 |
| Pi 5 + Hailo-8L | Hailo NPU | 640 | 30+ | 加速棒外挂 |
| RK3588 | NPU (RKNN) | 640 | 30-40 | 内置 NPU |

---

## 整体迁移路径

```
Windows/Mac PC               RK3588 板
+-----------------+         +-----------------+
| best.pt         |         |                 |
|     |           |         |                 |
|     v export    |         |                 |
| best.onnx       |         |                 |
|     |           |  scp    |                 |
|     v rknn-tk2  | ------> | best.rknn       |
| best.rknn       |         |     |           |
+-----------------+         |     v RKNNLite  |
                            | inference @NPU  |
                            +-----------------+
```

注意：**RKNN-Toolkit2** 必须在 **PC** 上做模型转换（用得是完整版 SDK，
依赖 PyTorch/TF 等）；**RKNNLite** 装在板子上用于推理。

---

## Step 1：在 PC 上把 .pt 转成 .onnx

```bash
# 在你训模型的电脑上
pip install ultralytics
python tools/export_ncnn.py models/best.pt --format onnx
# 产出: models/best.onnx
```

---

## Step 2：在 PC 上把 .onnx 转成 .rknn

```bash
pip install rknn-toolkit2  # PC 版
```

转换脚本（示例 `convert_to_rknn.py`，建议放到 `tools/`）：

```python
from rknn.api import RKNN

rknn = RKNN(verbose=True)

# RK3588 平台
rknn.config(
    mean_values=[[0, 0, 0]],
    std_values=[[255, 255, 255]],
    target_platform='rk3588',
    quantized_dtype='asymmetric_quantized-8',  # int8 量化
)

# 加载 ONNX
rknn.load_onnx(model='models/best.onnx')

# 量化（用一个目录的代表性图片做 calibration set）
rknn.build(do_quantization=True, dataset='dataset.txt')

# 导出
rknn.export_rknn('models/best.rknn')
rknn.release()
```

`dataset.txt` 是一个文本文件，每行一张代表性图片路径，建议用你训
集里随机选 50–100 张。

---

## Step 3：板子端运行环境

板子上的系统要求：Debian/Ubuntu + RK 提供的 NPU 内核驱动 + RKNNLite。

```bash
# 1. 安装 RKNNLite (Python)
pip install rknn-toolkit-lite2

# 2. 把 librknnrt.so 放到 /usr/lib（具体路径看你板子镜像，rk 出的镜像
#    通常已经装好了）
sudo find / -name librknnrt.so

# 3. 验证
python -c "from rknnlite.api import RKNNLite; print('ok')"
```

板子上的 `requirements.txt` 不需要 `ultralytics`，节省空间和启动时间。

---

## Step 4：把模型 scp 到板子

```bash
scp models/best.rknn pi@<rk3588-ip>:~/medical-waste-sorter/models/
```

---

## Step 5：写 RKNN 后端实现

`src/detector.py` 已经有 `RknnDetector` 的骨架，但 `predict()` 是
NotImplementedError。完整实现要包含：

1. **预处理**：letterbox 到 640×640，BGR→RGB，归一化
2. **NPU 推理**：`rknn.inference(inputs=[...])`
3. **后处理**：解码 YOLOv8 输出（grid + box+class score 拆分）→ NMS
4. **返回 Detection 列表**

可以参考官方仓库：
- https://github.com/airockchip/rknn_model_zoo/tree/main/examples/yolov8

把它们的 `yolov8.py` 后处理函数移植到我们的 `RknnDetector.predict()` 里
就能用。

---

## Step 6：切换 config

```yaml
detector:
  backend: rknn
  model_path: models/best.rknn
```

跑流水线和 Pi 上完全一样：

```bash
python -m src.main
```

## 性能调优

| 项 | 参数 | 推荐 |
|---|---|---|
| NPU 核 | RK3588 有 3 个 NPU 核 | 设 `core_mask=RKNNLite.NPU_CORE_0_1_2` 可并行 |
| 量化 | int8 | 比 fp16 快 2 倍，精度损失 1-2% |
| 输入分辨率 | 640 / 320 | 320 在 RK3588 上能 60+ FPS，但小目标精度下降 |
| 多线程 | 视觉/推理拆开 | 已经是双进程 |

## 常见问题

### 「Failed to load RKNN model」
- 模型是不是在 PC 上指定了 `target_platform='rk3588'`？指定成 `rk3568`
  或别的会加载失败。
- 量化时用的 toolkit 版本和板子上 `librknnrt.so` 版本要一致

### 「mean error very high after RKNN conversion」
- int8 量化损失过大，可以试 fp16：`quantized_dtype='asymmetric_quantized-16'`
  （或者干脆 `do_quantization=False`，但速度慢一倍）
- calibration set 不够代表性，加更多图

### 「inference 出来全 0 box」
- 后处理用的 anchor 或 stride 跟你训练用的不一致
- 训练时 input shape 不是 640，但导出时强制 640
